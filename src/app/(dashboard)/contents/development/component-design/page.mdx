# コンポーネント分割の設計

## この記事で学ぶこと

- コンポーネント分割の方針
- ファイル・フォルダ構成
- コンフリクト回避

---

## コンポーネント分割の方針

```
分割の基準：
├── 再利用できるか
├── 単一の責任を持つか
├── 適度なサイズか
└── 名前から役割がわかるか
```

---

## 実例：Todo アプリ

```
src/
├── components/
│   ├── Header.jsx
│   ├── TodoList.jsx
│   ├── TodoItem.jsx
│   ├── TodoForm.jsx
│   ├── TodoFilter.jsx
│   └── Modal.jsx
├── hooks/
│   ├── useTodos.js
│   └── useAuth.js
├── lib/
│   └── firebase.js
├── pages/
│   ├── Home.jsx
│   ├── Login.jsx
│   └── Register.jsx
└── App.jsx
```

---

## 役割の明確化

| コンポーネント | 役割 |
|:-------------|:----|
| Header | 共通ヘッダー |
| TodoList | Todo の一覧を表示 |
| TodoItem | 1つの Todo を表示 |
| TodoForm | Todo の入力フォーム |
| TodoFilter | フィルターボタン |

---

## コンフリクト回避

```
ポイント：
├── 担当者ごとにファイルを分ける
├── 同じファイルを同時に編集しない
├── 共通部分は早めに作る
└── こまめにマージする
```

---

## まとめ

この記事では、コンポーネント分割の設計を学びました。

- 分割の基準は **再利用性・単一責任・適度なサイズ・明確な名前**
- フォルダ構成を整理して**役割を明確化**する
- **コンフリクト回避**のためにファイルを分けて作業する
