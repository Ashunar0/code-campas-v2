# 要件定義の基礎

## この記事で学ぶこと

- 要件定義とは
- Why / Who / What
- Must と Want

---

## 要件定義とは

**要件定義**とは、「何を作るか」を明確にすることです。

```
要件定義で決めること：
├── なぜ作るのか（Why）
├── 誰が使うのか（Who）
├── 何を作るのか（What）
└── どこまで作るのか（Scope）
```

---

## Why（なぜ作るのか）

### 解決したい課題

```
例：
├── メモがバラバラで管理できない
├── 支出を把握できていない
├── 習慣が続かない
└── レシピを探すのが面倒
```

### 提供する価値

```
例：
├── メモを一元管理できる
├── 支出を可視化できる
├── 習慣を見える化して継続できる
└── 自分だけのレシピ集を作れる
```

---

## Who（誰が使うのか）

### ターゲットユーザー

```
例：
├── 大学生
├── 一人暮らしの社会人
├── 料理初心者
└── 自分自身
```

具体的にイメージできると設計がしやすくなります。

---

## What（何を作るのか）

### 機能一覧

```
例（習慣トラッカー）：
├── ユーザー登録・ログイン
├── 習慣の追加
├── 毎日のチェック
├── 達成状況の表示
└── 週間レポート
```

---

## Must と Want

### MoSCoW メソッド

**MoSCoW**（モスクワ）は、機能の優先度を分類するためのフレームワークです。

| 分類 | 意味 | MVP での扱い |
|:----|:----|:-----------|
| **Must** | 必須 | 実装する |
| **Should** | あった方がいい | 余裕があれば |
| **Could** | できれば | 後回し |
| **Won't** | 今回は作らない | 対象外 |

### MVP では Must のみ実装

```
Must（必須）：
├── ログイン機能
├── データの追加・表示
└── 基本的な操作

Want（欲しい）：
├── 検索機能
├── フィルター
└── グラフ表示
```

---

## 💡 Try: Why / Who / What を考えよう

作りたいアプリについて、以下を書き出してみましょう：

1. **Why**: どんな課題を解決する？
2. **Who**: 誰が使う？
3. **What**: どんな機能が必要？（Must を3つ挙げる）

---

## まとめ

この記事では、要件定義の基礎を学びました。

- 要件定義は「**何を作るか**」を明確にすること
- **Why（なぜ）/ Who（誰に）/ What（何を）** を整理する
- **MoSCoW メソッド**で優先度をつける
- MVP では **Must のみ**を実装する
