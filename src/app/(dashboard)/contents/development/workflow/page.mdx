
## この記事で学ぶこと

- 週ごとのゴール設定
- 進捗共有の方法
- 困ったときの対応

---

## 週ごとのゴール設定

```
Week 1: 設計 + 環境構築 + 認証
Week 2: CRUD の実装
Week 3: 追加機能 + スタイリング
Week 4: テスト + デプロイ + 発表準備
```

---

## 進捗共有の方法

### ツール

```
おすすめ：
├── Slack / Discord（日常のやり取り）
├── GitHub Issues / Projects（タスク管理）
└── 定期ミーティング（週1回）
```

### デイリースタンドアップ（任意）

```
毎日5分で共有：
├── 昨日やったこと
├── 今日やること
└── 困っていること
```

---

## 困ったときの対応

```
1. まず自分で調べる（15〜30分）
2. チームメンバーに相談
3. メンターに質問
4. 一緒に画面を見ながら解決
```

### ペアプログラミング

難しい問題は一緒に取り組むと解決が早いです。

---

## まとめ

この記事では、チーム開発の進め方を学びました。

- **週ごとのゴール**を設定して計画的に進める
- **Slack / Discord / GitHub** で進捗を共有する
- 困ったらまず自分で調べ、解決しなければ**チームに相談**する
