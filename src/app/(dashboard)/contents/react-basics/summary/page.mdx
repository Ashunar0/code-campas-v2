### 基本概念

| 項目           | 内容                                |
| -------------- | ----------------------------------- |
| JSX            | JavaScript の中に HTML のような構文 |
| コンポーネント | UI の部品、関数として定義           |
| Props          | 親から子へデータを渡す              |
| State          | コンポーネントが持つ動的なデータ    |

### フック

| フック     | 用途            |
| ---------- | --------------- |
| useState   | 状態を管理 ⭐   |
| useEffect  | 副作用を扱う ⭐ |
| useContext | Context を使う  |

### パターン

| パターン     | 内容                    |
| ------------ | ----------------------- |
| リスト描画   | map + key               |
| 条件分岐     | 三項演算子、&&          |
| フォーム     | Controlled Component    |
| API 取得     | useEffect + fetch       |
| リフトアップ | 共通の親に State を移動 |

## 特に重要なポイント

### 1. useState

```jsx
const [count, setCount] = useState(0);

// 更新は setter 関数を使う
setCount(count + 1);

// 配列・オブジェクトはスプレッド構文で
setTodos([...todos, newTodo]);
setUser({ ...user, name: "田中" });
```

### 2. useEffect

```jsx
// 初回のみ実行
useEffect(() => {
  fetchData();
}, []);

// 特定の値が変わったら実行
useEffect(() => {
  // valueが変わったとき
}, [value]);
```

### 3. Props と State の違い

|      | Props          | State      |
| ---- | -------------- | ---------- |
| 出所 | 親から渡される | 自分で持つ |
| 変更 | ❌ 不可        | ⭕ 可能    |

### 4. map でリスト描画

```jsx
{
  todos.map((todo) => <li key={todo.id}>{todo.text}</li>);
}
```

### 5. 配列操作

```jsx
// 追加
setTodos([...todos, newTodo]);

// 削除
setTodos(todos.filter((t) => t.id !== id));

// 更新
setTodos(todos.map((t) => (t.id === id ? { ...t, done: true } : t)));
```

## JavaScript 版と React 版の違い

| 観点       | JavaScript                | React                        |
| ---------- | ------------------------- | ---------------------------- |
| UI         | 命令的（手動で DOM 操作） | 宣言的（State を変えるだけ） |
| 状態       | 変数 + 手動更新           | useState                     |
| 表示と状態 | バラバラ                  | 一体化                       |

React では **State を変えるだけで画面が更新** されます！

## よくある質問

### Q: Props と State どっちを使えばいい？

**A**: データを使うコンポーネントで考えます。

- そのコンポーネントだけで使う → State
- 親から渡されたものを表示する → Props

### Q: useEffect がよくわからない

**A**: 最初は「**API 取得は useEffect の中**」と覚えれば OK。
依存配列は `[]`（初回のみ）から始めましょう。

### Q: コンポーネントをどう分割すればいい？

**A**: 「**1 つの役割**」を意識します。
迷ったら、まずは 1 ファイルで書いて、あとから分割しても大丈夫です。
