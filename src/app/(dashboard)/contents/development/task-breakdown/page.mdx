# タスク分割の基礎

## この記事で学ぶこと

- タスク分割とは
- 分割のポイント
- 実装順序の考え方

---

## タスク分割とは

大きな機能を、小さな作業単位に分解することです。

```
悪い例：
└── Todo アプリを作る（巨大すぎる）

良い例：
├── 環境構築
├── Firebase 設定
├── ログイン機能
├── Todo 一覧表示
├── Todo 追加
├── Todo 完了切り替え
└── ...
```

---

## 分割のポイント

### 1つのタスクは数時間〜1日以内

```
目安：
├── 小さすぎ: 30分未満 → まとめる
├── 適切: 2〜4時間
└── 大きすぎ: 1日以上 → 分割する
```

### 完了条件を明確に

```
例：
├── 「ログイン画面が表示される」
├── 「メールとパスワードでログインできる」
└── 「エラーメッセージが表示される」
```

---

## 実装順序を考える

### 依存関係を意識

```
1. 環境構築 ← 最初に必要
2. Firebase 設定
3. 認証機能  ← 先に作る（他が依存）
4. データ取得・表示
5. データ追加
6. データ編集・削除
7. スタイリング ← 最後でOK
```

---

## 優先度をつける

Must の機能から実装します。

```
Week 1: 環境構築 + 認証
Week 2: CRUD の基本
Week 3: 追加機能 + テスト + デプロイ
```

---

## 💡 Try: タスクを分割してみよう

あなたのアプリの **1つの機能**（例：ログイン機能）を、5つのタスクに分割してみましょう。

例：
1. Firebase Auth の設定
2. ログインフォームの UI 作成
3. ログイン処理の実装
4. エラーハンドリング
5. ログイン後のリダイレクト

---

## まとめ

この記事では、タスク分割の基礎を学びました。

- タスク分割は**大きな機能を小さな作業単位に分解**すること
- 1タスクは **2〜4時間** が目安
- **完了条件を明確に**する
- **依存関係を意識**して実装順序を決める
