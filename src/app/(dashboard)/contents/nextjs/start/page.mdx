## Next.js とは

**Next.js** は、React をベースにした Web アプリケーションフレームワークです。

```
Next.js の特徴：
├── ファイルベースルーティング
├── サーバーサイドレンダリング（SSR）
├── 静的サイト生成（SSG）
├── API Routes
├── 画像最適化
└── 自動コード分割
```

---

## React との違い

> 💡 **用語解説**
> - **SEO**（Search Engine Optimization）: 検索エンジン最適化。Googleなどで上位表示されやすくする施策
> - **CSR**（Client Side Rendering）: ブラウザ側でHTMLを生成する方式
> - **SSR**（Server Side Rendering）: サーバー側でHTMLを生成する方式

| 項目 | React（Vite） | Next.js |
|:----|:-------------|:--------|
| ルーティング | 自分で設定（React Router） | ファイルで自動設定 |
| レンダリング | クライアントのみ（CSR） | サーバー + クライアント |
| SEO | 弱い | 強い（SSR/SSG） |
| セットアップ | シンプル | やや複雑 |
| デプロイ | Firebase Hosting など | Vercel が最適 |

---

## なぜ Next.js を学ぶのか

### メリット

```
✅ SEO に強い（サーバーサイドレンダリング）
✅ ルーティングが簡単（ファイルで定義）
✅ パフォーマンスが良い（自動最適化）
✅ フルスタック開発が可能（API Routes）
✅ 業界で広く採用されている
```

### 注意点

```
⚠️ React より複雑
⚠️ 学習コストがある
⚠️ サーバーが必要（一部機能）
⚠️ MVP では必須ではない
```

---

## MVP で Next.js を使うかどうか

### React のみで十分な場合

```
├── シンプルな単一ページアプリ
├── SEO が重要でない
├── Firebase Hosting でデプロイ
└── チームが React に慣れていない
```

### Next.js を検討する場合

```
├── 複数ページのサイト
├── SEO が重要（ブログ、LP など）
├── サーバーサイドの処理が必要
└── Vercel でデプロイしたい
```
