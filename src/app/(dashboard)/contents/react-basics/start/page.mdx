## React とは何か

React は、**ユーザーインターフェース（UI）** を作るための JavaScript ライブラリです。

特徴：

- **コンポーネント指向**：画面を部品（コンポーネント）に分けて作る
- **宣言的 UI**：「こう表示したい」と書くだけで React が描画してくれる
- **一方向データフロー**：データの流れがわかりやすい

世界中の多くの企業で使われていて、Web 開発の標準的なスキルになっています。

## なぜ React を学ぶのか

JavaScript 章で作ったアプリ（クイズ、メモなど）を思い出してください。

JavaScript では：

- `querySelector` で要素を取得
- `innerHTML` で中身を変更
- データが変わるたびに手動で DOM を更新

**React では**：

- データ（State）を変えるだけで**自動的に画面が更新**される
- コードがシンプルで読みやすい
- 再利用しやすいコンポーネントが作れる

この違いを、この章で実感しましょう！

## 躓きやすいポイントと対策

React は**最も躓きやすい章**です。
以下のポイントを意識しながら学習しましょう。

### 1. State と Props の違い

|          | State        | Props                            |
| -------- | ------------ | -------------------------------- |
| 誰が持つ | 自分自身     | 親から渡される                   |
| 変更可能 | ⭕           | ❌                               |
| 用途     | 動的なデータ | コンポーネント間のデータ受け渡し |

### 2. State の更新方法

```jsx
// ❌ 直接変更しない
count = count + 1;

// ⭕ setter 関数を使う
setCount(count + 1);
```

### 3. 配列・オブジェクトの State 更新

```jsx
// ❌ 直接変更しない
todos.push(newTodo);
setTodos(todos);

// ⭕ 新しい配列を作って渡す
setTodos([...todos, newTodo]);
```

### 4. useEffect の依存配列

```jsx
// 毎レンダリング実行（避ける）
useEffect(() => { ... });

// 初回のみ実行
useEffect(() => { ... }, []);

// 特定の値が変わったら実行
useEffect(() => { ... }, [count]);
```
