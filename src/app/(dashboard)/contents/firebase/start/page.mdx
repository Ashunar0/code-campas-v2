## この章について

ついに **Firebase** を使った本格的なアプリ開発に入ります！🔥

これまで作ったアプリは、データがブラウザ（localStorage）にのみ保存されていました。
Firebase を使うと、**クラウドにデータを保存**し、**ユーザー認証**も実装できます。

## Firebase とは？

**Firebase** は、Google が提供する **BaaS（Backend as a Service）** です。

> 💡 **BaaS とは？**
>
> BaaS（Backend as a Service）は、アプリのバックエンド機能をクラウドで提供するサービスです。
> サーバーを自分で構築・運用する必要がなく、API を呼び出すだけで認証やデータ保存などの機能が使えます。

自分でサーバーを構築しなくても、以下の機能が使えます：

| サービス           | 機能                     |
| ------------------ | ------------------------ |
| **Authentication** | ユーザー認証（ログイン） |
| **Firestore**      | データベース             |
| **Storage**        | ファイル保存（画像など） |
| **Hosting**        | Web サイトのホスティング |
| **Functions**      | サーバーレス関数         |

## なぜ Firebase を使うのか

### 従来の開発

```
フロントエンド（React）
    ↓ API 通信
バックエンド（Node.js, Python など）
    ↓ SQL クエリ
データベース（MySQL, PostgreSQL など）
```

- サーバーの構築・運用が必要
- セキュリティ対策が必要
- 開発コストが高い

### Firebase を使った開発

```
フロントエンド（React）
    ↓ Firebase SDK
Firebase（認証・DB・ストレージ）
```

- サーバー不要
- セキュリティは Firebase 側で対応
- 開発スピードが速い
