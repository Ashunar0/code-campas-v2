- どんなアプリを作るか

---

## どんなアプリを作るか

### アイデアの源泉

```
身近な課題から考える：
├── 自分が困っていること
├── 友人が困っていること
├── 日常の不便なこと
└── もっと便利にしたいこと

既存サービスの簡易版：
├── メモアプリ
├── 家計簿アプリ
├── 習慣トラッカー
└── レシピ管理
```

---

## ブレインストーミング

### やり方

1. 時間を決める（15分など）
2. 思いついたアイデアを全部書き出す
3. 批判しない
4. 量を重視
5. 後から整理・評価

### チームでのアイデア出し

```
流れ：
├── 各自で5個アイデアを出す
├── 全員で共有
├── グルーピング
├── 投票で絞り込む
└── 1つに決定
```

---

## MVP に適したアイデアの条件

### 3週間で作れる規模

```
✅ 3〜5画面程度
✅ 基本的な CRUD
✅ 認証機能
✅ シンプルな UI

❌ 複数のユーザー間のリアルタイム連携
❌ 決済機能
❌ 複雑なアルゴリズム
```

> 💡 **CRUD** とは、Create（作成）・Read（読み取り）・Update（更新）・Delete（削除）の頭文字で、データ操作の基本4機能を指します。

### 明確なユーザー価値

```
良い例：
├── 「○○が便利になる」
├── 「○○の手間が省ける」
└── 「○○が見える化できる」

悪い例：
├── 「機能がたくさんある」
├── 「技術的にすごい」
└── 「作りたいから」
```

## 💡 Try: アイデアを出してみよう

1. **身近な課題**を3つ書き出してみましょう
2. その中から**アプリで解決できそうなもの**を1つ選びましょう
3. そのアプリの**ユーザー価値**（「○○が△△になる」）を一文で書いてみましょう

---

## 実現可能性のチェック

```
確認すること：
├── 必要な機能は学んだ技術で作れるか
├── 外部 API が必要な場合、無料で使えるか
├── チームのスキルで対応できるか
└── 期間内に完成できるか
```

---

## まとめ

この記事では、アプリのアイデアの出し方を学びました。

- アイデアは**身近な課題**から考えると見つけやすい
- **ブレインストーミング**で量を出し、後から絞り込む
- MVP に適したアイデアは **3〜5画面程度** の規模
- **明確なユーザー価値**があるアイデアを選ぶ
