## React とは？

**React** は、UI（ユーザーインターフェース）を作るための JavaScript ライブラリです。

- Facebook（現 Meta）が開発
- 2013 年にオープンソース化
- 世界中で最も使われている UI ライブラリの一つ

## コンポーネント指向

React の最大の特徴は **コンポーネント指向** です。

画面を小さな「部品（コンポーネント）」に分割して作ります。

```
┌─────────────────────────────────────┐
│ Header（ヘッダーコンポーネント）       │
├─────────────────────────────────────┤
│ Sidebar │ Main                      │
│         │ ┌───────────────────────┐ │
│         │ │ Card  Card  Card      │ │
│         │ └───────────────────────┘ │
├─────────────────────────────────────┤
│ Footer（フッターコンポーネント）       │
└─────────────────────────────────────┘
```

### メリット

1. **再利用性** - 同じコンポーネントを何度も使える
2. **保守性** - 一箇所を修正すれば全体に反映
3. **分かりやすさ** - 役割ごとにファイルが分かれる

## 宣言的 UI

React は **宣言的** に UI を記述します。

### 命令的（JavaScript）

「どうやって」表示するかを記述：

```javascript
// 要素を取得して、中身を変えて、スタイルを変えて...
const element = document.getElementById("counter");
element.textContent = count;
element.style.color = count < 0 ? "red" : "black";
```

### 宣言的（React）

「何を」表示するかを記述：

```jsx
// こう表示したい、と書くだけ
<p style={{ color: count < 0 ? "red" : "black" }}>{count}</p>
```

React が「どうやって」表示するかは自動で処理してくれます。

## 仮想 DOM

React は **仮想 DOM（Virtual DOM）** という仕組みを使っています。

### 通常の DOM 操作

DOM を直接操作すると、変更のたびにブラウザが再描画されて遅くなることがあります。

### 仮想 DOM の仕組み

1. React はメモリ上に「仮想 DOM」を作る
2. State が変わると、新しい仮想 DOM を作る
3. 新旧の仮想 DOM を比較（差分検出）
4. 変更があった部分だけ実際の DOM を更新

これにより、**効率的な描画** が可能になります。

> 💡 仮想 DOM の詳細を深く理解する必要はありません。
> 「React が効率的に画面を更新してくれる」と覚えておけば OK！

## 一方向データフロー

React では、データは **親から子へ** 一方向に流れます。

```
親コンポーネント（State を持つ）
    ↓ Props でデータを渡す
子コンポーネント
```

この流れにより、データの変更が追跡しやすくなります。

## Vite + React プロジェクトの確認

Chapter 0 で作成した React プロジェクトを使います。

まだ作っていない場合は、以下のコマンドで作成：

```bash
cd ~/Desktop/webex
npm create vite@latest react-practice -- --template react
cd react-practice
npm install
npm run dev
```

### プロジェクト構造（復習）

```
react-practice/
├── node_modules/
├── public/
├── src/
│   ├── App.jsx          ← メインのコンポーネント
│   ├── main.jsx         ← エントリーポイント
│   ├── App.css
│   └── index.css
├── index.html
├── package.json
└── vite.config.js
```

### App.jsx を見てみよう

```jsx
function App() {
  return (
    <div>
      <h1>Hello React!</h1>
    </div>
  );
}

export default App;
```

これが **コンポーネント** です。
関数の形で書かれていて、**JSX** を返しています。

## React の 3 つの特徴まとめ

| 特徴               | 説明                     |
| ------------------ | ------------------------ |
| コンポーネント指向 | UI を部品に分けて作る    |
| 宣言的 UI          | 「何を表示するか」を書く |
| 仮想 DOM           | 効率的な画面更新         |

## まとめ

- **React** は UI を作るための JavaScript ライブラリ
- **コンポーネント** という部品を組み合わせて画面を作る
- **宣言的** に「何を表示するか」を書く
- **仮想 DOM** で効率的に画面を更新
- データは **親から子へ** 一方向に流れる
