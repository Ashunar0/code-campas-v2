## Docker とは

**Docker** は、アプリケーションをコンテナとしてパッケージ化するツールです。

```
メリット：
├── 環境の差異がなくなる
├── どこでも同じ環境で動く
├── チームで環境を統一
└── 本番環境との差がなくなる
```

---

## コンテナとイメージ

```
イメージ: 設計図（テンプレート）
コンテナ: 実行中のインスタンス

例：
├── node:18 (イメージ)
└── my-app (コンテナ)
```

---

## 基本的なコマンド

```bash
# イメージの取得
docker pull node:18

# コンテナの起動
docker run -it node:18

# コンテナ一覧
docker ps
docker ps -a  # 停止中も含む

# コンテナの停止
docker stop <container_id>

# イメージ一覧
docker images
```

---

## なぜ Docker を使うのか

**具体例**: チームメンバーの PC で `npm install` が失敗する...

```
Docker なし：
├── 人によって Node.js のバージョンが違う
├── Mac と Windows で動作が違う
└── 「自分の環境では動く」問題

Docker あり：
├── 全員同じ環境で開発
├── 設定ファイルを共有するだけ
└── 新メンバーもすぐ開始
```

---

## まとめ

- **Docker** はアプリケーションをコンテナとしてパッケージ化
- **イメージ**は設計図、**コンテナ**は実行中のインスタンス
- `docker pull` でイメージ取得、`docker run` でコンテナ起動
- 環境の統一とチーム開発に効果的

---

## Try：Docker コマンドを試そう

1. Docker がインストールされていることを確認
2. Node.js のイメージを取得して起動

```bash
# Docker バージョン確認
docker --version

# Node.js イメージを取得
docker pull node:18

# コンテナ内で Node.js を実行
docker run -it node:18 node -v
```
