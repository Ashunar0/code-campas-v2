> - Git のインストール
> - 初期設定（user.name, user.email）
> - GitHub アカウントの作成

## Git とは？

**Git** は、ファイルの変更履歴を管理するツールです。
「**バージョン管理システム**」とも呼ばれます。

例えるなら、ゲームの「**セーブ機能**」のようなもの。

- いつでも過去の状態に戻れる
- 誰が、いつ、何を変更したかがわかる
- 複数人で同時に開発できる

Web 開発では、Git を使うのが当たり前になっています。

- 「セーブポイント」のような点が並んでいる
-->

## GitHub とは？

**GitHub** は、Git で管理しているコードを**インターネット上に保存**できるサービスです。

Git と GitHub の関係：
- **Git**：あなたの PC 上でファイルを管理
- **GitHub**：その管理しているファイルをインターネットに保存・共有

GitHub を使うと：
- コードをバックアップできる
- チームでコードを共有できる
- 自分の作品を公開できる（ポートフォリオにもなる）

## Git のインストール

### Mac の場合

Mac には Git が初めからインストールされていることが多いです。
まずは確認してみましょう。

```bash
git --version
```

バージョンが表示されれば OK です。

```
git version 2.39.0
```

表示されない場合は、以下の方法でインストール：

**方法 1: Xcode Command Line Tools**
```bash
xcode-select --install
```

**方法 2: Homebrew**（Homebrew が入っている場合）
```bash
brew install git
```

### Windows の場合

1. [Git for Windows](https://git-scm.com/download/win) にアクセス

2. 「Click here to download」をクリック

<!-- 
📸 画像メモ: Git for Windows のダウンロードページ
-->

3. ダウンロードした `.exe` を実行

4. 基本的にはデフォルトのまま「Next」で進めて OK

<!-- 
📸 画像メモ: Git for Windows のインストーラー
- 基本的には Next を押していく
-->

5. インストール完了後、Git Bash を開いて確認

```bash
git --version
```

## Git の初期設定

Git をインストールしたら、**自分の名前とメールアドレス**を設定します。
これは、「誰がコードを変更したか」を記録するためです。

ターミナルで以下のコマンドを実行：

```bash
# 名前を設定（自分の名前に置き換えてね）
git config --global user.name "あなたの名前"

# メールアドレスを設定（自分のメアドに置き換えてね）
git config --global user.email "your-email@example.com"
```

> 💡 **GitHub アカウントのメールアドレスと同じものを設定するのがおすすめです**

設定を確認するには：

```bash
git config --global user.name
git config --global user.email
```

-->

## GitHub アカウントの作成

### 1. GitHub にアクセス

[GitHub](https://github.com/) にアクセスします。

### 2. Sign up をクリック

<!-- 
📸 画像メモ: GitHub トップページ
- Sign up ボタンを赤枠で囲む
-->

### 3. 必要な情報を入力

- **Email**: メールアドレス
- **Password**: パスワード（強力なものを設定）
- **Username**: ユーザー名（公開されるので、覚えやすいものを）

<!-- 
📸 画像メモ: GitHub サインアップ画面
-->

### 4. 確認メールをチェック

登録したメールアドレスに確認メールが届くので、リンクをクリック。

### 5. 登録完了！

GitHub のダッシュボードが表示されれば OK です。

<!-- 
📸 画像メモ: GitHub ダッシュボード
- 登録直後のダッシュボード画面
-->

## GitHub との接続確認

GitHub アカウントを作成したら、自分の PC（Git）と GitHub が正しく接続できるか確認しましょう。

### HTTPS での接続（簡単）

Git の操作時に、GitHub のユーザー名とパスワード（またはトークン）を入力する方式です。

初心者はまずこの方式で OK。詳しい設定は Chapter 6（Git / GitHub の章）で行います。

### SSH での接続（推奨・任意）

より安全で便利な接続方式です。
一度設定すれば、パスワードを毎回入力する必要がなくなります。

> 💡 SSH の設定は時間がかかるので、初回面談では省略しても OK。
> 後で Chapter 6 で詳しく説明します。

## まとめ

- **Git** はファイルの変更履歴を管理するツール
- **GitHub** は Git で管理したコードをインターネット上に保存するサービス
- Git をインストールしたら、**user.name** と **user.email** を設定
- **GitHub アカウント**を作成して、オンラインでコードを管理
