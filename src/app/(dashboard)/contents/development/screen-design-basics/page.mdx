## ワイヤーフレームとは

**ワイヤーフレーム**は、画面のレイアウトを簡単なスケッチで表現したものです。

```
特徴：
├── 色やデザインは含まない
├── 配置と構造を決める
├── 手書きでも OK
└── チームで認識を合わせる
```

## なぜ画面設計が必要か

```
メリット：
├── 完成形をイメージできる
├── チームで認識を合わせられる
├── 実装前に問題に気づける
├── 開発スピードが上がる
└── 手戻りが減る
```

## ツールの紹介

| ツール | 特徴 | 難易度 |
|:------|:----|:-----:|
| **Figma** | 無料でプロ向け | ★★☆ |
| **Excalidraw** | 手書き風でシンプル | ★☆☆ |
| 紙とペン | 最も手軽 | ★☆☆ |

### MVP では紙 + Figma がおすすめ

```
流れ：
├── 紙に手書きでラフを描く
├── チームで確認
├── Figma で清書（任意）
```

---

## ワイヤーフレームに含めるもの

```
含めるもの：
├── 画面のレイアウト
├── ヘッダー / フッター
├── ボタン、フォームの配置
├── リストの表示イメージ
├── モーダルのイメージ

含めないもの：
├── 色
├── 細かいデザイン
├── 実際のデータ
```

## 💡 Try: ワイヤーフレームを描いてみよう

紙とペンを使って、あなたのアプリの**ホーム画面**のワイヤーフレームを描いてみましょう。

- ヘッダー、メインコンテンツ、フッターの配置を決める
- ボタンやフォームの位置を四角で表現する
