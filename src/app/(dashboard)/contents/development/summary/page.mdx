# まとめと MVP 開発へ

## この章で学んだこと

Chapter 8 では、開発プロセス（要件定義・設計・タスク分割）を学びました。

---

## 習得したスキル

```
✅ 開発フローの全体像
✅ 要件定義の方法
✅ 画面設計（ワイヤーフレーム）
✅ データ設計（Firestore 構造）
✅ タスク分割と Issue 管理
✅ チームでのルール決め
```

---

## MVP 開発の流れ

```
Week 1: 企画・設計
├── アイデア決め
├── 要件定義
├── 画面設計
├── データ設計
└── タスク分割

Week 2-3: 実装
├── 環境構築
├── 認証機能
├── CRUD 機能
└── スタイリング

Week 4: 仕上げ
├── テスト
├── デプロイ
└── 発表準備
```

---

## チーム開発での心構え

```
大切なこと：
├── コミュニケーション第一
├── 困ったらすぐ相談
├── 完璧より「動くもの」
├── こまめにコミット・マージ
└── 楽しもう！
```
