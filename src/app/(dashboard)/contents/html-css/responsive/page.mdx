## レスポンシブデザインとは？

**レスポンシブデザイン**とは、画面サイズに応じてレイアウトが変化する Web デザインのことです。

スマホ、タブレット、PC など、様々なデバイスで見やすい表示を実現します。

{/*
📸 画像メモ: レスポンシブデザインのイメージ
- 同じサイトがスマホ、タブレット、PC で表示されている
- それぞれに適したレイアウト
*/}

## viewport の設定

レスポンシブデザインを有効にするためには、HTML の `<head>` 内に以下の設定が必要です：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

- `width=device-width`：画面幅をデバイスの幅に合わせる
- `initial-scale=1.0`：初期のズームレベルを 1（100%）に

> 💡 Vite で作った React プロジェクトには、この設定は最初から入っています。

## メディアクエリ

**メディアクエリ**を使うと、画面サイズに応じて異なる CSS を適用できます。

### 基本的な書き方

```css
/* 通常のスタイル（すべての画面サイズに適用） */
.container {
  width: 100%;
}

/* 画面幅が 768px 以下のとき */
@media (max-width: 768px) {
  .container {
    padding: 10px;
  }
}
```

### min-width と max-width

| 書き方             | 意味                 |
| ------------------ | -------------------- |
| `max-width: 768px` | 768px **以下**のとき |
| `min-width: 768px` | 768px **以上**のとき |

```css
/* 768px 以下（スマホ・タブレット） */
@media (max-width: 768px) {
  /* スマホ用スタイル */
}

/* 769px 以上（PC） */
@media (min-width: 769px) {
  /* PC用スタイル */
}
```

## ブレークポイント

画面サイズの「区切り」をブレークポイントと呼びます。
よく使われる値：

| デバイス   | ブレークポイント |
| ---------- | ---------------- |
| スマホ     | 〜 480px         |
| タブレット | 481px 〜 768px   |
| PC（小）   | 769px 〜 1024px  |
| PC（大）   | 1025px 〜        |

```css
/* スマホ */
@media (max-width: 480px) {
  /* ... */
}

/* タブレット */
@media (max-width: 768px) {
  /* ... */
}

/* PC */
@media (min-width: 1025px) {
  /* ... */
}
```

> 💡 数値は厳密なルールはありません。プロジェクトによって調整します。

## モバイルファースト設計

**モバイルファースト**とは、まずモバイル用のスタイルを書き、そこから画面が大きくなるにつれてスタイルを追加していく設計方法です。

### デスクトップファースト（従来）

```css
/* PC用（デフォルト） */
.container {
  width: 1200px;
}

/* タブレット */
@media (max-width: 768px) {
  .container {
    width: 100%;
  }
}

/* スマホ */
@media (max-width: 480px) {
  .container {
    padding: 10px;
  }
}
```

### モバイルファースト（推奨）

```css
/* モバイル用（デフォルト） */
.container {
  width: 100%;
  padding: 10px;
}

/* タブレット以上 */
@media (min-width: 481px) {
  .container {
    padding: 20px;
  }
}

/* PC以上 */
@media (min-width: 769px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }
}
```

> 💡 モバイルファーストが推奨される理由：
>
> - モバイルユーザーが多い
> - シンプルな状態から複雑な状態へ拡張しやすい

## 実践例

### 1. ナビゲーションの切り替え

```css
/* モバイル：縦並び */
.nav {
  display: flex;
  flex-direction: column;
  gap: 10px;
}

/* PC：横並び */
@media (min-width: 769px) {
  .nav {
    flex-direction: row;
    gap: 30px;
  }
}
```

### 2. カードのレイアウト

```css
.card-grid {
  display: grid;
  grid-template-columns: 1fr; /* モバイル：1列 */
  gap: 20px;
}

@media (min-width: 481px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr); /* タブレット：2列 */
  }
}

@media (min-width: 769px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr); /* PC：3列 */
  }
}
```

### 3. フォントサイズの調整

```css
h1 {
  font-size: 24px; /* モバイル */
}

@media (min-width: 769px) {
  h1 {
    font-size: 36px; /* PC */
  }
}
```

## レスポンシブで気をつけること

### 1. 画像のサイズ

```css
img {
  max-width: 100%; /* 親要素からはみ出さない */
  height: auto; /* 縦横比を維持 */
}
```

### 2. 横スクロールを防ぐ

```css
body {
  overflow-x: hidden; /* 横スクロールを無効化 */
}

.container {
  width: 100%; /* 100% を超えないように */
  max-width: 1200px;
}
```

### 3. タップしやすいボタンサイズ

```css
button {
  min-width: 44px; /* 最小幅 */
  min-height: 44px; /* 最小高さ */
  padding: 12px 24px;
}
```

## まとめ

- **レスポンシブデザイン**で様々な画面サイズに対応
- **viewport** の設定が必須
- **メディアクエリ**で画面サイズに応じたスタイルを適用
- **モバイルファースト**で書くのがおすすめ
- **開発者ツール**でテストする
