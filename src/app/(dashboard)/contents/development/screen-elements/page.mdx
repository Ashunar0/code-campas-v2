# 画面ごとの要素を洗い出す

## この記事で学ぶこと

- 各画面に何が必要か
- 実例で考える
- コンポーネント分割を意識

---

## 各画面に何が必要か

各画面に必要な要素をリストアップします。

```
画面名: Todo一覧
要素:
├── ヘッダー（ロゴ、ログアウトボタン）
├── 検索フォーム
├── フィルター（全て/完了/未完了）
├── Todoリスト
│   └── Todoアイテム × N
├── 追加ボタン（FAB）
└── フッター（任意）
```

---

## 実例：Todo 一覧ページ

```
Todo一覧ページ
├── Header
│   ├── ロゴ
│   └── LogoutButton
├── SearchForm
│   └── 検索入力欄
├── FilterButtons
│   ├── 全て
│   ├── 完了
│   └── 未完了
├── TodoList
│   └── TodoItem
│       ├── チェックボックス
│       ├── テキスト
│       ├── 編集ボタン
│       └── 削除ボタン
└── AddButton（追加ボタン）
```

> 💡 **FAB（Floating Action Button）** とは、画面の右下などに浮かんで表示される丸いボタンのことです。追加アクションによく使われます。

---

## 💡 Try: 画面の要素を洗い出そう

あなたのアプリの**メイン画面**について、上の例を参考にツリー形式で要素を洗い出してみましょう。

- Header、Main、Footer の3つのエリアに分けると整理しやすいです

---

## コンポーネント分割を意識

```
components/
├── Header.jsx
├── SearchForm.jsx
├── FilterButtons.jsx
├── TodoList.jsx
├── TodoItem.jsx
└── AddButton.jsx
```

要素をリストアップすることで、必要なコンポーネントが見えてきます。

---

## State はどこに持つか

```
TodoList で管理する State:
├── todos（Todo の配列）
├── filter（フィルター状態）
└── searchQuery（検索クエリ）
```

設計段階で State の場所を考えておくと、実装がスムーズになります。

---

## まとめ

この記事では、画面ごとの要素の洗い出し方を学びました。

- 各画面に必要な要素を**ツリー形式で**洗い出す
- 要素を洗い出すと**必要なコンポーネント**が見えてくる
- 設計段階で **State の場所**を考えておく
