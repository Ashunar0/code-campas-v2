---

## テストとは

アプリが正しく動作するか確認することです。

```
テストの種類：
├── 手動テスト（人が操作して確認）
└── 自動テスト（コードでテスト）

MVP では手動テストで十分です。
```

---

## テスト項目のチェックリスト

### 機能テスト

```
認証：
├── [ ] 新規登録できる
├── [ ] ログインできる
├── [ ] ログアウトできる
└── [ ] 未ログインでアクセスするとリダイレクト

CRUD：
├── [ ] データを追加できる
├── [ ] データが表示される
├── [ ] データを編集できる
└── [ ] データを削除できる
```

### エラーハンドリング

```
├── [ ] 空欄で送信するとエラー表示
├── [ ] 不正な入力でエラー表示
└── [ ] 通信エラーでエラー表示
```

### UI/UX

```
├── [ ] ローディング表示がある
├── [ ] レスポンシブ対応
└── [ ] 操作後にフィードバックがある
```

---

## デバッグの方法

```
ツール：
├── console.log（基本）
├── React DevTools
├── Chrome DevTools
└── Firebase Console
```

---

## まとめ

この記事では、テストの基礎を学びました。

- MVP では **手動テスト** で十分
- **機能テスト・エラーハンドリング・UI/UX** をチェックリストで確認
- デバッグには **console.log** や **DevTools** を活用
