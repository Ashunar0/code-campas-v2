## Git / GitHub とは

### Git とは

**Git** は、ファイルの変更履歴を記録・管理するための**バージョン管理システム**です。

```
Git でできること：
├── 変更履歴の記録
├── 過去の状態への復元
├── 誰が何を変更したかの追跡
└── 複数人での同時開発
```

### GitHub とは

**GitHub** は、Git で管理しているプロジェクトを**クラウド上で保管・共有**できるサービスです。

```
GitHub でできること：
├── コードのバックアップ
├── チームメンバーとの共有
├── Pull Request でのコードレビュー
├── Issue でのタスク管理
└── プロジェクトの公開（ポートフォリオ）
```

## なぜバージョン管理が必要か

### バージョン管理なしの問題

バージョン管理を使わないと、こんな問題が起きます：

```
プロジェクトフォルダ/
├── index_最終版.html
├── index_最終版2.html
├── index_最終版_修正.html
├── index_最終版_修正2_田中確認済.html
└── index_これが本当の最終版.html  ← どれが最新？
```

### バージョン管理ありの場合

Git を使えば、1つのファイルで履歴を管理できます：

```
$ git log --oneline

a1b2c3d ヘッダーのデザインを修正
e4f5g6h フッターを追加
i7j8k9l 初期コミット
```

- どの時点でも**復元可能**
- **誰が**変更したかわかる
- ファイルが増えない

## チーム開発での重要性

### 1人開発でも重要

```
✅ コードのバックアップ
✅ 変更履歴の記録
✅ 実験的な変更を安全に試せる
✅ ポートフォリオとして公開できる
```

### チーム開発では必須

```
✅ 複数人で同時に作業できる
✅ 変更を統合（マージ）できる
✅ コードレビューができる
✅ 誰が何をしたか追跡できる
```

## MVP 開発と Git

MVP 開発期間では、Git / GitHub を実践的に使います。

### チームでの活用

```
開発フロー：
1. Issue でタスクを作成
2. ブランチを切って開発
3. こまめにコミット
4. Pull Request を作成
5. レビューを受けてマージ
```

### 就職活動での活用

```
GitHub プロフィール：
├── 制作物のソースコード
├── コミット履歴（継続的に開発している証拠）
├── README.md（プロジェクト説明）
└── 草（コントリビューション）
```
