# 要件定義書を書こう

## この記事で学ぶこと

- 要件定義書のテンプレート
- 実例で学ぶ

---

## 要件定義書のテンプレート

```markdown
# アプリ名

## Why（目的）

### 解決したい課題
- 課題1
- 課題2

### 提供する価値
- 価値1
- 価値2

## Who（ターゲットユーザー）

- ユーザー像の説明

## What（機能）

### Must（必須機能）
- [ ] 機能1
- [ ] 機能2
- [ ] 機能3

### Want（あると良い機能）
- [ ] 機能4
- [ ] 機能5

## 技術スタック

- React + Vite
- Tailwind CSS
- Firebase (Auth, Firestore)
```

---

## 実例：Todo アプリ

```markdown
# シンプル Todo

## Why（目的）

### 解決したい課題
- タスクを忘れてしまう
- 優先度がわからなくなる

### 提供する価値
- タスクを一覧で管理できる
- 完了状況が一目でわかる

## Who（ターゲットユーザー）

- タスク管理をシンプルにしたい人

## What（機能）

### Must（必須機能）
- [ ] ログイン / ログアウト
- [ ] タスクの追加
- [ ] タスクの表示
- [ ] タスクの完了切り替え
- [ ] タスクの削除

### Want（あると良い機能）
- [ ] タスクの編集
- [ ] 検索機能
- [ ] フィルター（完了/未完了）
```

---

## チームで合意を取る

要件定義書は **チーム全員で確認** し、認識を合わせましょう。

```
確認ポイント：
├── 目的は共有できているか
├── 機能の範囲は妥当か
├── MVP で作る範囲は明確か
└── 期間内に完成できそうか
```

---

## 💡 Try: 要件定義書を書いてみよう

上のテンプレートを使って、自分のアプリの要件定義書を書いてみましょう。

- まずは **Must を3〜5個** 挙げることから始めましょう
- Want は「余裕があれば」と割り切って、欲張らないのがコツです

---

## まとめ

この記事では、要件定義書の書き方を学びました。

- テンプレートを使うと**迷わず書ける**
- **Why / Who / What + 技術スタック**を記載する
- **チーム全員で確認**して認識を合わせる
