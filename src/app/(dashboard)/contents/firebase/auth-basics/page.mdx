## Firebase Auth とは？

**Firebase Authentication** は、ユーザー認証を簡単に実装できるサービスです。

> 💡 **認証と認可の違い**
>
> - **認証（Authentication）**: 「あなたは誰？」→ ユーザーの身元を確認すること
> - **認可（Authorization）**: 「何ができる？」→ ユーザーの権限を判定すること
>
> Firebase Auth は主に **認証** を担当します。認可はセキュリティルールで行います。

- メール・パスワード認証
- Google / Twitter / GitHub ログイン
- 電話番号認証
- 匿名認証

## 認証方法の種類

| 認証方法              | 説明                           |
| --------------------- | ------------------------------ |
| **メール/パスワード** | 最も基本的な方法 ⭐            |
| Google                | Google アカウントでログイン    |
| GitHub                | GitHub アカウントでログイン    |
| 匿名                  | 登録なしで仮ユーザーとして利用 |

この章では**メール/パスワード認証**を実装します。

## メール/パスワード認証を有効化

1. Firebase Console を開く
2. **「Authentication」** をクリック
3. **「Sign-in method」** タブを開く
4. **「メール/パスワード」** をクリック
5. **「有効にする」** をオン
6. **「保存」**

## 認証の UI フロー

### サインアップ（ユーザー登録）

```
1. ユーザーがメール・パスワードを入力
2. createUserWithEmailAndPassword で登録
3. 成功 → ホーム画面へ遷移
4. 失敗 → エラーメッセージを表示
```

### ログイン

```
1. ユーザーがメール・パスワードを入力
2. signInWithEmailAndPassword でログイン
3. 成功 → ホーム画面へ遷移
4. 失敗 → エラーメッセージを表示
```

### ログアウト

```
1. signOut を呼び出す
2. ログイン画面へ遷移
```
