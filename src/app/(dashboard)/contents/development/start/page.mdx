# はじめに

## この章で学ぶこと

Chapter 8 では、**開発プロセス**（要件定義・設計・タスク分割）を学びます。

### 主な学習内容

1. **開発フロー** - Web アプリ開発の全体像
2. **要件定義** - 何を作るか明確にする
3. **画面設計** - ワイヤーフレームの作成
4. **データ設計** - Firestore の構造を考える
5. **タスク分割** - 実装を細かく分ける

---

## なぜ開発プロセスが重要か

### 「作りながら考える」の限界

```
よくある失敗：
├── 途中で何を作るかわからなくなる
├── チームで認識がズレる
├── 何度も作り直しになる
├── 期限に間に合わない
└── 完成しても使えないものになる
```

### 設計してから作るメリット

```
✅ ゴールが明確になる
✅ チームで認識を合わせられる
✅ 手戻りが減る
✅ 計画的に進められる
✅ 品質が上がる
```

---

## MVP 開発に向けての準備

この章は **MVP 開発の準備** として最も重要です。

```
MVP 開発の流れ：
├── Week 1: 企画・設計 ← この章の内容
├── Week 2-3: 実装
└── Week 4: テスト・デプロイ・発表
```

---

## この章の使い方

### チームで一緒に進める

設計は一人で進めるのではなく、チームで話し合いながら進めましょう。

### テンプレートを活用する

この章では多くのテンプレートを用意しています。そのまま使ってOKです。

### 完璧を目指さない

最初から完璧な設計は不要です。作りながら修正していけばOKです。
