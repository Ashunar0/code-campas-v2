## 重要なコマンド一覧

### 基本操作

| コマンド | 説明 |
|:--------|:----|
| `git init` | リポジトリ初期化 |
| `git status` | 状態確認 |
| `git add .` | 全ファイルをステージング |
| `git commit -m "msg"` | コミット |
| `git log --oneline` | 履歴確認（1行表示） |
| `git diff` | 差分確認 |

### リモート操作

| コマンド | 説明 |
|:--------|:----|
| `git push` | リモートに送信 |
| `git pull` | リモートから取得 |
| `git clone URL` | リポジトリをコピー |
| `git remote -v` | リモート確認 |

### ブランチ操作

| コマンド | 説明 |
|:--------|:----|
| `git branch` | ブランチ一覧 |
| `git checkout -b name` | ブランチ作成・切替 |
| `git switch -c name` | ブランチ作成・切替（新構文） |
| `git merge branch` | マージ |
| `git branch -d name` | ブランチ削除 |

### 取り消し・修正

| コマンド | 説明 |
|:--------|:----|
| `git stash` | 一時保存 |
| `git stash pop` | 一時保存を復元 |
| `git reset --soft HEAD~1` | コミット取り消し（変更残る） |
| `git revert コミットID` | コミット打ち消し |
| `git commit --amend` | 直前のコミット修正 |

## よくある質問

### Q: コミットを間違えた場合は？

**A:** `git reset --soft HEAD~1` で直前のコミットを取り消せます。変更はステージングに残ります。

### Q: プッシュした後に修正したい場合は？

**A:** 新しいコミットで修正してプッシュしましょう。履歴が残るので問題ありません。

### Q: .env をプッシュしてしまった場合は？

**A:** 重大なセキュリティ問題です。以下の手順で対応してください：
1. すぐにチームに報告
2. API キーを無効化・再発行
3. 履歴から削除（`git filter-branch`）
4. `.gitignore` に追加

### Q: コンフリクトが怖いです

**A:** コンフリクトは「チームで同時に作業している証拠」です。手順に従えば必ず解決できます。怖がらずに経験を積みましょう。

### Q: どのくらいの頻度でコミットすべき？

**A:** 「意味のある単位」でこまめにコミットしましょう。目安は1〜2時間に1回以上。大きな変更を1つにまとめるより、小さく分けた方が良いです。
